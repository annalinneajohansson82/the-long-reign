# Purpose

Define what The Long Reign is, at the highest level. This document is the single-sentence summary every design, system, and feature must support. When in doubt about a decision, return here first.

---

# Overview

**The Long Reign** is an original browser-based kingdom-building simulation game. It is not a clone of any existing game.

The player begins with a handful of settlers and an untouched frontier. Over many hours, they build:

- a capital
- colonies
- roads
- industries
- trade networks
- communities
- history

The primary emotional goal is that a player returns after months away, loads their save, looks over their kingdom, and thinks:

> "Wow… I built all this."

The settlement—not combat, not heroes—is the heart of the game. The kingdom is the protagonist.

---

# The North Star

Every design decision should make the kingdom feel more alive.

Not more complicated. Not more difficult. More alive.

This principle overrides individual mechanic preferences whenever uncertainty exists.

---

# Project Identity

> The Long Reign is a game about creating a kingdom worth returning to—not because the game demands it, but because the player genuinely wants to see it again.

---

# Design Goals

- Build a kingdom that feels alive over generations.
- Prioritize the settlement as the central subject of play.
- Reward long-term attachment: the kingdom should improve each time the player returns.
- Create a persistent world the player is proud to have shaped.

---

# Non-Goals

- The game is not a browser clone of TopHeroes or any other existing title. The original inspiration served only as a starting point.
- The game is not an idle hero RPG. Combat supports settlement progression, not the other way around.

---

# Player Experience

The player should feel like the steward of a living, breathing world—not a manager optimizing a spreadsheet. Their kingdom grows visibly, develops identity, and accumulates history. Time away from the game should deepen appreciation, not create pressure to catch up.
